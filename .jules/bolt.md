## 2026-05-24 - Array allocation optimizations
**Learning:** Found multiple instances where proxy arrays (`ColumnarDataset` outputs or large `allClientsData`) were iterated multiple times via chained `.filter()` calls. This caused massive intermediate array allocations resulting in high memory pressure and FPS drops, especially on mobile devices.
**Action:** Replaced chained `array.filter(cond1).filter(cond2)` sequences with single-pass `for` loops that evaluate `cond1` and `cond2` simultaneously and push directly into a pre-allocated/locally-scoped array. Also limited `.slice()` during heavy map-to-innerHTML table renders from 500 to 50 items per initial paint to avoid blocking the main thread for long periods.
